Create a production-grade, fully offline, single-file HTML application called "IT Project Manager Workspace" for a project manager managing multiple IT projects. The final output must be ONE standalone .html file with all CSS and JavaScript embedded. Do not use external CDN files, external JavaScript, external CSS, proprietary corporate templates, or company-specific data structures. The design must be modern, sleek, clean, highly responsive, and easy to use.

Primary objective:
Build one PM workspace that combines Fixed Bid/Waterfall project control, Agile Scrum/T&M execution, Hybrid delivery traceability, budget tracking, team/resource management, RAID management, project-level reporting, all-project summary reporting, and offline AI-style predictive analytics.

User persona:
The tool is exclusively for an IT Project Manager. Do not add role-switching dropdowns such as Delivery Lead, PMO, Portfolio Lead, Program Manager, etc. The PM may manage multiple projects of different operating models.

Project operating models:
1. Fixed Bid / Waterfall: Use this when deliverables, timelines, cost, scope and acceptance criteria are mostly fixed. It must enable WBS task creation, dynamic Gantt, MS Project-style task hierarchy, dependencies, baseline, critical path and earned value style controls.
2. Agile Scrum / Time & Material: Use this when scope evolves sprint-by-sprint. It must enable product backlog, sprint creation, user stories, story points, sprint board, Kanban columns, burndown, burnup and velocity.
3. Hybrid: Enable both fixed WBS/Gantt and Scrum board. Add a Hybrid Links section to connect Scrum stories to fixed WBS deliverables.

Navigation and UX:
- Use a top project dropdown to select the active project.
- Keep all-project summary separate from selected-project sections.
- Use grouped dropdown menus, not a long horizontal menu that overflows the screen.
- Suggested menu groups:
  - All Projects: Summary, Register
  - Project Work: Cockpit, Fixed Plan/Gantt, Sprint Board, Hybrid Links, Budget, Team, RAID, Reports, AI Prediction
  - Setup & Guide: Masters, Help, Appendix, Code Whitepaper
- Project-scoped sections must be disabled or show a friendly prompt if no project is selected.
- Gantt is available only for Fixed Bid/Waterfall and Hybrid projects.
- Sprint Board is available only for Agile Scrum/T&M and Hybrid projects.
- Hybrid Links is available only for Hybrid projects.
- Use light positive theme by default. Add theme options: Light Positive, Mint Calm, Techno Light, Dark Command, Hi-Tech Dark.
- Use compact typography, about 1px smaller than default enterprise dashboards.
- Use clean cards, rounded corners, soft shadows, white/light gradients, blue/teal/green accents.
- Support responsive layouts for desktop, tablet and mobile.
- Add tooltips and help icons for formulas, report meanings and AI scores.
- Add a floating quick calculator.
- Add global search for projects, tasks, stories and risks.
- Add backup and restore controls.

Fixed Bid / Waterfall module:
Implement a dynamic Gantt chart driven by an automated scheduling engine, not a static timeline. It must include:
- Split-screen WBS task table on the left and timeline graph on the right.
- WBS nesting with summary tasks, normal tasks and milestones.
- Custom fields: WBS, task name, type, duration, start, finish, percent complete, assignee, predecessor string, task cost and notes.
- Task creation and editing modals.
- Dependencies: Finish-to-Start, Start-to-Start, Finish-to-Finish and Start-to-Finish.
- Lead and lag syntax such as 1.2FS+2, 2.1SS+0, 3FF-1, 4SF+3.
- Automatic recalculation of successor dates when a predecessor date/duration changes.
- CPM logic: forward pass, backward pass, early start/finish, late start/finish, total float.
- Highlight zero-float critical path tasks in red.
- Baseline save and baseline comparison. Show saved baseline bars underneath live bars.
- Show today line.
- Show dependency connector lines and dependency labels.
- Allow basic horizontal drag of live bars to shift task dates, then recalculate downstream tasks.
- Add PDF/print export for the selected-project Gantt.

Implement a Scrum board with:
- Product backlog.
- Sprint creation and active sprint selection.
- User story creation with epic, story title, points, priority, status, assignee, blocker flag and optional WBS link.
- Kanban workflow: Backlog, To Do, In Progress, In Review, Done.
- Drag-and-drop story cards between columns.
- Story points.
- Sprint goal fields if possible.
- Burndown chart.
- Burnup chart.
- Velocity chart.
- Cumulative flow chart.
- Blocker and aging indicators.
- Sprint failure risk prediction.

Hybrid module:

- Add a Hybrid Links view that links stories to WBS deliverables or milestones.
- Show which fixed deliverables are supported by which agile stories.
- Use traceability reports and warnings when fixed deliverables have no linked agile implementation stories.

Budget module:
Selected-project only. Support:
- Budget categories from master data.
- CAPEX/OPEX style category tagging where possible.
- Planned, actual and forecast cost.
- Resource cost, license cost, infrastructure/cloud cost, vendor cost, travel, training and contingency.
- Task-level planned/actual cost for WBS tasks.
- Budget summary KPIs.
- EVM style calculations: BAC, PV, EV, AC, SPI, CPI, SV, CV, EAC, ETC, VAC.
- Budget waterfall chart.
- Planned vs actual vs forecast charts.

Team/resource module:

- Project-specific team selection.
- Resource allocation, utilization and capacity views.
- Quick-add resource from task/story creation, without forcing the PM to leave the work planning flow.
- Resource cost charts, utilization charts, bubble charts and overload alerts.
